¿Por qué es una buena práctica usar JavaScript para checar que sean válidos los inputs de las formas antes de enviar los datos al servidor? Si la información está en el formato correcto, la aplicación permite que los datos se envíen al servidor y (en general) que se guarden en una base de datos; si la información no está en el formato correcto, da al usuario un mensaje de error que explica lo que debe corregir y le permite volver a intentarlo. Fuente: Mozilla. (2023). Validación de formularios. MDN Web Docs. Recuperado el 11 de abril de 2024, de https://developer.mozilla.org/es/docs/Learn/Forms/Form_validation

Cómo puedes saltarte la seguridad de validaciones hechas con JavaScript? Al paracer sí, se puede saltar la seguridad de las validaciones hechas con JavaScript utilizando herramientas de desarrollo del navegador que permiten modificar el código JavaScript en tiempo real. Además, los usuarios avanzados pueden deshabilitar JavaScript en su navegador, lo que evitará que se ejecuten las validaciones. Fuente: ADA. (09/12/2020). Cómo saltarse la validación HTML5 de un formulario. ADA. Recuperado el 11 de abril de 2024, de https://didacticode.com/como-saltarse-validacion-html5-formulario

Si te puedes saltar la seguridad de las validaciones de JavaScript, entonces ¿por qué la primera pregunta dice que es una buena práctica? Aunque es posible saltarse las validaciones de JavaScript, es una buena práctica utilizarlas porque proporcionan una capa adicional de validación para la mayoría de los usuarios que no intentarán evadir las validaciones. Además, las validaciones del lado del cliente pueden detectar errores de forma rápida y permitir correcciones instantáneas sin tener que esperar la respuesta del servidor. Fuente: Mozilla. (2023).
