Preguntas a responder

¿Cuál es la diferencia entre Internet y la World Wide Web? R. Internet es una inmensa red de computadoras alrededor de todo el mundo conectadas entre sí. En cambio, World Wide Web es una enorme colección de páginas que se asienta sobre esa red de computadoras. En comparativa el sartén y la carne respectivamente.

¿Cuáles son las partes de un URL? R. Protocolo. Subdominio. Dominio. TLD. Carpeta o subcarpeta. Página. Etiqueta.

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE? R. GET solicita una representación de un recurso específico. Las peticiones que usan el método GET solo deben recuperar datos. HEAD pide una respuesta idéntica a la de una petición GET, pero sin el cuerpo de la respuesta. POST envía una entidad a un recurso en específico, causando a menudo un cambio en el estado o efectos secundarios en el servidor. PATCH aplica modificaciones parciales a un recurso. PUT reemplaza todas las representaciones actuales del recurso de destino con la carga útil de la petición. DELETE borra un recurso en específico.

¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué? R. El método HTTP más comúnmente utilizado al enviar un formulario HTML es el método POST. La razón principal es que el método POST envía los datos del formulario en el cuerpo de la solicitud HTTP, lo que proporciona mayor seguridad y privacidad en comparación con el método GET.

¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL? R. Cuando accedes a una página web a través de un URL en tu navegador web, se utiliza el método HTTP GET de forma implícita. El método GET se utiliza para solicitar datos de un recurso específico en un servidor. Cuando escribes la URL en la barra de direcciones del navegador y presionas Enter, el navegador envía una solicitud GET al servidor para recuperar la página web correspondiente.

Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error? R. Cuando un servidor web devuelve una respuesta HTTP con el código 200, significa que la solicitud se ha procesado correctamente y que la respuesta contiene el recurso solicitado. El código de estado HTTP 200 se conoce como "OK" o "Éxito" y es una indicación de que la operación fue exitosa.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué? R. Claro. Esto ocurre porque un error 404 indica que el servidor no pudo encontrar la página solicitada. Es importante que se asegure de que todas las páginas del sitio web sean accesibles.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué? R. También. Este error 500 indica un error interno del servidor, lo que significa que ha ocurrido un problema en el servidor que impide que se complete la solicitud del usuario. Este tipo de error generalmente se debe a un problema en el lado del servidor.

¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? R. Significa que se considera obsoleto o anticuado, y se recomienda evitar su uso para que se usen las alternativas más modernas.

Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados. R. applet Relacionada con applets de Java. acronym Para insertar acrónimos. bgsound Servía para insertar sonidos de fondo. dir Lista de archivos o carpetas. hgroup Agrupación de encabezados.

¿Cuáles son las diferencias principales entre HTML 4 y HTML5? R. El HTML sólo utiliza la caché del navegador y las cookies para almacenar datos temporalmente. HTML5 utiliza las bases de datos SQL de la web, el almacenamiento local y la caché de la aplicación para almacenar los datos temporalmente. El JavaScript y la interfaz del navegador se ejecutan en el mismo hilo.

¿Qué componentes de estructura y estilo tiene una tabla? R. Encabezamientos de las tablas. Celdas de datos de las tablas. Títulos de las tablas.

¿Cuáles son los principales controles de una forma HTML5? R. En HTML 5 se añadieron los controles: botón general , número , búsqueda , teléfono , dirección web , dirección de correo , fecha , hora , color , rango , salida y barra de progreso .

Con ciclo de vida se refiere a la duración de cada una de las fases que este conlleva. En la siguiente pregunta se mencionan las etapas. Pero este ciclo de vida es parecido a un ciclo de vida de la vida real. ¿Cuál es el ciclo de desarrollo de sistemas de información? R.
